🔐 Security Features
Role-Based Access Control
Roles:

USER - Standard user access
ADMIN - Full administrative access
MODERATOR - Moderate user management
Permissions:

READ_USERS - View user information
WRITE_USERS - Create and update users
DELETE_USERS - Delete users
READ_ROLES - View roles
MANAGE_ROLES - Manage role assignments
READ_PERMISSIONS - View permissions
MANAGE_PERMISSIONS - Manage permissions
Method-Level Security Examples
Java

@PreAuthorize("hasRole('ADMIN')")
public List<UserDTO> getAllUsers() { ... }

@PreAuthorize("hasAnyRole('USER', 'ADMIN')")
public UserDTO getCurrentUser() { ... }

@PreAuthorize("isAuthenticated()")
public String getUserProfile() { ... }

🔄 Token Refresh Flow
text

1. User logs in → Receives access token (15 min) + refresh token (7 days)
2. Access token expires → Use refresh token to get new access token
3. Refresh token expires → User must log in again
4. Logout → Refresh token is revoked
5. Daily cleanup → Expired tokens removed at 2 AM

🔍 Troubleshooting
Common Issues
1. Token Expired Error:

Use the refresh token endpoint to get a new access token
If refresh token is also expired, login again
2. Access Denied:

Check if user has required role/permission
Verify token is included in Authorization header
3. H2 Console Not Accessible:

Ensure spring.h2.console.enabled=true
Check URL: http://localhost:8080/h2-console
4. Database Not Initialized:

Verify spring.jpa.defer-datasource-initialization=true
Check data.sql is in resources folder
